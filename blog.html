Frequently Asked Questions

Before ES6, everyone used to with Var. To delcare a variable. But there was some problem with Var. Any variable declared with without function scope can be accessed and can be re-declared. In ES6 LET and var introduced. As LET only accessible within the block and function scope and LET can be updated but not re-declared, all issues with VAR now solved. CONST is kind of LET but its cannot be updated or re-declared.

There is significant difference between Normal function and Arrow function. The difference is not only in surface level also in the underlying level how they works. We all know that Arrow function is introduced in ES6 and it has supority over normal function. If our expression and parameter is single, we can omit () {} also we can omit return keyword. But arrow function is not hoisted meaning that we cant call arrow function before initiation it. Also they deal with THIS keyword differently.

Template Literals is one of the importan feature of ES6. It also solved many problems with string. If we use tempate literals we can dynamically add items inside it. Besides that, we can use as much as space and line in template literals. Before that we have to concat with + and \n etc.
